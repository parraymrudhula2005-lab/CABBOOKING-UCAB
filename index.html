# 🚕 Cab-U Frontend A modern React-based frontend for the Cab-U booking application. ## Features - ✅ User authentication (Login/Register) - ✅ Responsive design - ✅ Book rides easily - ✅ Ride history tracking - ✅ Driver registration - ✅ Driver dashboard - ✅ Real-time location updates - ✅ Rating system ## Tech Stack - **Framework**: React 18 - **Build Tool**: Vite - **State Management**: Zustand - **HTTP Client**: Axios - **Routing**: React Router v6 - **CSS**: Custom CSS (responsive) ## Installation ### 1. Install Dependencies ```bash npm install ``` ### 2. Create .env file Create `.env` in the frontend root directory: ```env VITE_API_URL=http://localhost:5000/api ``` ### 3. Start Development Server ```bash npm run dev ``` The frontend will open at `http://localhost:3000` ## Available Routes ### Public Routes - `/` - Home page - `/login` - User login - `/register` - User registration ### Protected Routes (Requires Login) - `/dashboard` - User dashboard - `/book-ride` - Book a new ride - `/ride-history` - View past rides - `/driver-register` - Register as driver - `/driver-dashboard` - Driver dashboard ## Project Structure ``` frontend/ ├── src/ │ ├── components/ │ │ └── Navbar.jsx (Navigation component) │ ├── pages/ │ │ ├── Home.jsx (Home page) │ │ ├── Login.jsx (Login page) │ │ ├── Register.jsx (Registration page) │ │ ├── Dashboard.jsx (User dashboard) │ │ ├── BookRide.jsx (Book ride page) │ │ ├── RideHistory.jsx (Ride history) │ │ ├── DriverRegister.jsx (Driver registration) │ │ └── DriverDashboard.jsx (Driver dashboard) │ ├── services/ │ │ └── api.js (API service with axios) │ ├── context/ │ │ └── authStore.js (Zustand auth store) │ ├── styles/ │ │ ├── index.css (Global styles) │ │ ├── navbar.css (Navbar styles) │ │ ├── auth.css (Auth pages styles) │ │ ├── home.css (Home page styles) │ │ ├── dashboard.css (Dashboard styles) │ │ ├── book-ride.css (Book ride styles) │ │ ├── ride-history.css (History styles) │ │ ├── driver-register.css (Driver register styles) │ │ └── driver-dashboard.css (Driver dashboard styles) │ ├── App.jsx (Main app component) │ └── main.jsx (App entry point) ├── public/ (Static assets) ├── index.html (HTML template) ├── vite.config.js (Vite config) ├── package.json (Dependencies) └── .env (Environment variables) ``` ## Build for Production ```bash npm run build ``` Output will be in the `dist/` folder. ## Preview Production Build ```bash npm run preview ``` ## API Integration All API calls are handled through the `api.js` service which automatically: - Adds JWT token to requests - Handles authentication - Manages base URL configuration ### API Endpoints Used ```javascript // Users userAPI.register(userData) userAPI.login(credentials) userAPI.getProfile() userAPI.updateProfile(data) // Drivers driverAPI.register(driverData) driverAPI.getProfile() driverAPI.updateLocation(location) driverAPI.toggleOnline() driverAPI.getNearby(location) driverAPI.getEarnings() // Bookings bookingAPI.requestRide(bookingData) bookingAPI.acceptRide(bookingId) bookingAPI.getBooking(bookingId) bookingAPI.updateStatus(bookingId, status) bookingAPI.getUserBookings() bookingAPI.completeRide(bookingId, rating, review) ``` ## State Management Uses Zustand for authentication state: ```javascript import { useAuthStore } from './context/authStore' // In components: const { token, user, login, logout, register } = useAuthStore() ``` ## Features Breakdown ### Authentication - User registration with validation - Secure login with JWT - Automatic token refresh - Logout functionality - Protected routes ### Booking System - Easy ride booking interface - Real-time ride tracking - Ride history view - Rating and review system ### Driver Features - Driver registration with document upload - Online/offline toggle - Location updates - Earnings dashboard - Ride request management ## Responsive Design - Mobile-first approach - Breakpoints: 768px (tablet), 480px (mobile) - Touch-friendly buttons - Optimized layouts for all screen sizes ## Error Handling - User-friendly error messages - Form validation - API error handling - Loading states ## Security - JWT token stored in localStorage - Automatic token inclusion in requests - Protected API routes - Input validation - CORS support ## Performance - Code splitting with React.lazy - Optimized re-renders with Zustand - Efficient API calls - CSS optimization - Image optimization ready ## Troubleshooting ### Port 3000 Already in Use ```bash # Change port in vite.config.js # Or kill process using port 3000 ``` ### API Connection Error 1. Ensure backend is running: `npm run dev` in backend folder 2. Check `VITE_API_URL` in `.env` 3. Verify CORS is enabled in backend ### Blank Page After Login 1. Check browser console for errors 2. Verify token is stored in localStorage 3. Check Redux/Zustand state ## Next Steps 1. Run `npm install` 2. Create `.env` file 3. Ensure backend is running 4. Run `npm run dev` 5.
